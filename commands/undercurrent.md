---
description: 潛流主題發掘
arguments: [範圍] — 可選，無則掃描全部
---

先執行 _init.md 中的 vault 偵測與資料夾解析，取得 vault 路徑和 analysis、daily-notes、inbox 資料夾位置。同時取得所有內容資料夾清單。

掃描 vault 找出潛流主題，並將結果存入分析資料夾。

輸入：$ARGUMENTS（可選。無參數則掃描全部。有參數則聚焦特定範圍）

掃描範圍（無參數時）：
- 所有內容資料夾（深度不限）
- daily-notes 資料夾全部
- inbox 資料夾全部

執行邏輯：
1. 檢查 {analysis}/undercurrent/ 中今天是否已有檔案
2. 如果有 → 追加模式
3. 如果沒有 → 建立模式

【建立模式】

寫入目標：{analysis}/undercurrent/YYYY-MM-DD.md
如果資料夾不存在，先建立並印出完整路徑。

檔案結構：

```
# 潛流分析

> 分析時間：YYYY-MM-DD HH:MM
> 掃描範圍：[列出實際掃描的目錄]

## 潛流主題

### 1. {主題/詞彙}
- 出現次數：{N} 次，涉及 {M} 個檔案
- 出現脈絡：
  - [[檔名A]]：[該檔案中的使用脈絡，一句話]
  - [[檔名B]]：[同上]
  （上限 5 個檔案）
- 判斷：[這個主題值得展開嗎？為什麼？]

### 2. {主題/詞彙}
...
```

判斷標準：
- 出現在 3 個以上不同檔案中
- 沒有以它為主題的獨立文章或 draft
- 不是常見功能詞或結構詞

【追加模式】

在既有檔案末尾追加：

```
---

## 追蹤更新 [HH:MM]

### 新發現的潛流
- [今天掃描發現的、之前未列出的潛流主題，格式同上]

### 已有潛流的變化
- [之前列出的潛流主題，出現頻率或脈絡有無變化]
```

規則：
- 按出現頻率倒序排列
- 上限 10 個潛流主題
- 所有檔案引用使用 [[wiki link]] 格式
- 時間戳使用本地時間 HH:MM 格式（24 小時制）
- 寫入前顯示即將寫入的檔名、模式（建立/追加）、內容摘要和完整路徑，等確認後再執行
- 寫入後印出實際寫入的完整路徑
- 同時在終端輸出完整分析結果

以 _init.md 中偵測到的語言撰寫。
