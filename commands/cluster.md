先執行 _init.md 中的 vault 偵測與資料夾解析，取得 vault 路徑和 analysis、daily-notes、inbox 資料夾位置。同時取得所有內容資料夾清單。

分析 vault 中正在聚攏成形的想法群，並將結果存入分析資料夾。

輸入：$ARGUMENTS（可選。無參數則掃描全部。有參數則聚焦特定範圍）

掃描範圍（無參數時）：
- 所有內容資料夾
- daily-notes 資料夾最近 30 天
- inbox 資料夾全部

執行邏輯：
1. 檢查 {analysis}/cluster/ 中今天是否已有檔案
2. 如果有 → 追加模式
3. 如果沒有 → 建立模式

【建立模式】

寫入目標：{analysis}/cluster/YYYY-MM-DD.md
如果資料夾不存在，先建立並印出完整路徑。

檔案結構：

```
# 想法群聚攏分析

> 分析時間：YYYY-MM-DD HH:MM
> 掃描範圍：[列出實際掃描的目錄]

## 想法群

### 1. {建議的群組名稱}
- 涉及筆記：
  - [[檔名A]]（修改日期：YYYY-MM-DD）
  - [[檔名B]]（修改日期：YYYY-MM-DD）
- 共同主題：[2 到 3 句摘要]
- 成熟度：高 / 中 / 低
- 建議發展方向：文章 / 專案 / 概念框架 / 繼續累積

### 2. {建議的群組名稱}
...
```

判斷標準：
- 3 篇以上筆記涉及相似概念但沒有共同的上層分類
- 這些筆記的撰寫時間分布在不同日期（不是一次性產出）

【追加模式】

在既有檔案末尾追加：

```
---

## 追蹤更新 [HH:MM]

### 新發現的想法群
- [今天掃描發現的、之前未列出的想法群，格式同上]

### 已有想法群的變化
- [之前列出的想法群，是否有新筆記加入、成熟度是否提升]
```

規則：
- 上限 5 個想法群，按成熟度倒序
- 所有檔案引用使用 [[wiki link]] 格式
- 時間戳使用本地時間 HH:MM 格式（24 小時制）
- 寫入前顯示即將寫入的檔名、模式（建立/追加）、內容摘要和完整路徑，等確認後再執行
- 寫入後印出實際寫入的完整路徑
- 同時在終端輸出完整分析結果

以 _init.md 中偵測到的語言撰寫。
