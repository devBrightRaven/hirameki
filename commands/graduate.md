description: 想法畢業為獨立筆記
---

從 `~/.claude/CLAUDE.md` 讀取 `## Vault Structure`，取得 vault 路徑和 daily-notes、logs、inbox 資料夾位置。同時取得所有內容資料夾清單。
若不存在或缺少必要欄位，停止並回應：「尚未完成初始設定，請先執行 `/hirameki:__init`」

掃描範圍：
- daily-notes 資料夾最近 14 天（從 Wrap 區塊中找反覆出現在「下一步」但未執行的、「進行中」卡住的）
- logs 資料夾最近 14 天（從 Journal 中找靈感連結、改進方向、未完成項目）
- inbox 資料夾全部
- 所有內容資料夾下遞迴搜尋名稱匹配 draft* 或 thought* 的子目錄（不分大小寫）

目標：找出值得畢業為獨立筆記的半成形想法。

判斷標準：
- 有明確的核心主張（不只是隨想或問句）
- 與 vault 中至少一個既有主題相關
- 有足夠的內容密度（不只是一句話）

輸出分兩階段：

【第一階段：候選清單】
列出符合條件的想法，每筆包含：
- 來源檔案與位置（哪個段落）
- 核心主張摘要（一句話）
- 建議歸屬的內容資料夾
- 與哪些既有筆記相關

列出候選後暫停，等我確認哪些要畢業。

【第二階段：執行畢業】（等確認後才執行）
對每個確認的想法，在對應的內容資料夾下建立新的 markdown 檔案，包含：
- 標題
- 核心主張
- 出處脈絡（從哪個 daily note 或 inbox 項目發展來的）
- 與 vault 其他筆記的關聯（wiki link 格式）
- 待展開的方向

寫入前顯示即將建立的檔案完整路徑，等確認後再執行。
寫入後印出實際建立的完整路徑。

以 __init.md 中偵測到的語言撰寫。
